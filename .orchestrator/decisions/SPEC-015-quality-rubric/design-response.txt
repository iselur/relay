Requiring `quality` makes it part of the acceptance gate structurally, despite being semantically advisory. That is acceptable fail-closed behavior only if explicitly acknowledged, deployed atomically with the v3 prompt/schema, and covered by retry/fallback behavior. “Quality never gates” is inaccurate; valid presence gates, scores do not.

Material findings:

- The same reviewer producing binary judgment and quality scores creates anchoring/halo risk. Prompt wording cannot ensure the scores do not influence the verdict. Fix: obtain quality in a second pass after freezing the binary verdict, ideally a separate invocation; otherwise document this residual coupling.
- One all-scores-1 test is insufficient. Add metamorphic tests proving that, for identical binary fields, every valid score/evidence combination yields the same gate result. Keep gate evaluation in a function whose inputs exclude `quality`.
- Protect deployment/versioning: require an explicit `schema_version: 3`, atomically update prompt/schema/parser, distinguish historical v2 from malformed current output, and prove in-flight/retry behavior. Never silently treat a current missing block as historical v2.
- Scores are ordinal; averaging 1–5 implies questionable precision. Prefer anchored enums such as `POOR|ADEQUATE|GOOD|EXCELLENT`, or define behavioral anchors for every numeric score. Three dimensions are a reasonable minimal set, but `maintainability` and `design_fit` need non-overlapping definitions.
- Require evidence containing non-whitespace text and ideally a concrete diff/test citation. A bare `minLength: 1` accepts whitespace and boilerplate.
- Future leakage remains possible through thresholds, ranking, dashboards, or human approval pressure. Add a policy/test forbidding quality-derived merge eligibility, exit status, retry, prioritization, or reviewer verdict; label metrics advisory and emit dimension distributions plus coverage, not only averages.
- A sandboxed worker does not remove insider-style logic risk in `dispatch.py`. Require focused tests for schema downgrade/confusion, malformed quality, v2 metrics handling, score invariance, and unchanged binary rejection cases; independently inspect the exact gate diff.

VERDICT: BLOCK
