## (1) Enforcement mechanisms at task intake

Make delegation and planning an intake state machine, not a preference. Add a mandatory “first-action reflex” to `CLAUDE.md`:

1. Classify the request as non-substantive, ordinary delegated, high-assurance delegated, or control-plane.
2. For every substantive request, invoke the Codex planning wrapper before analysis, drafting, editing, research, or implementation.
3. Record the returned plan ID and lane.
4. Challenge and authorize the plan.
5. Delegate execution to Codex unless the lane explicitly reserves the action.
6. At completion, compare the delivered work with the authorized plan and record deviations.

Create `scripts/codex-plan`, a low-friction wrapper around detached `codex exec -c service_tier=priority`. It accepts the task, constraints, proposed lane, affected paths, and acceptance criteria; emits a structured plan artifact plus JSON event stream; and recovers the last `agent_message` if the normal output file is absent. It must follow the existing no-minute-timeout rule. A `--small` mode changes expected plan length, not the requirement to plan.

Enforce prerequisites mechanically:

- `scripts/dispatch.py` rejects a spec without an authorized Codex-authored `plan_ref` and matching plan digest.
- CI rejects changes to control-plane paths—dispatcher, gates, schemas, workflows, `CLAUDE.md`, isolation, approval logic, or delegation policy—unless the change references an authorized decision record and plan.
- The session ritual requires Claude to state internally: lane, plan ID, delegated executor, and reserved Claude action before substantive work.
- `scripts/delegation_report.py` ingests plan, authorization, dispatch, execution, review, and exception events. Unclassified work counts as Claude work, preventing favorable reporting through omission.
- Any bypass requires a human-created, reasoned, expiring exception artifact. Claude cannot self-grant one.

Add two checkpoints to `CLAUDE.md`: before execution, “Is Codex carrying every delegable step?”; before responding, “Did execution follow the authorized plan, and are deviations recorded?”

## (2) Codex-plans-everything workflow

The ownership chain is:

**Codex drafts → Claude challenges → Codex disposes/revises → Claude authorizes → Codex executes → deterministic gates test → Claude independently reviews → human merges or performs reserved action.**

Claude’s challenge must identify at least one of: unsupported assumption, missing failure mode, scope ambiguity, insufficient test, or reason none applies. “Looks good” is not a valid challenge. Codex then returns a disposition: revise, reject with evidence, or accept and amend. Authorization records the final digest, so later silent plan edits invalidate dispatch.

Store artifacts according to purpose:

- `.orchestrator/plans/PLAN-NNN.md`: tracked source plan for every substantive task, with task, lane, author, assumptions, ordered actions, tests, rollback, challenge, disposition, authorization, and final status.
- `specs/SPEC-NNN.yaml`: executable contract for dispatched work, referencing the authorized plan and digest. Approved specs remain immutable.
- `.orchestrator/decisions/ADR-NNN.md`: durable control-plane or policy decision, referencing the plan; it records the authority decision, not a duplicate implementation plan.
- Per-attempt evidence records the plan/spec digests actually used.

The only planless fast path is a request that is both read-only and non-substantive: greetings, clarification questions, locating a known file, reporting already-observed status, or a single deterministic lookup requiring no interpretation. It cannot edit state, recommend a decision, synthesize research, draft user-facing substantive text, diagnose a problem, or trigger external action. If uncertain, plan. Tiny corrections use `codex-plan --small`, producing a five-field micro-plan: objective, scope, action, verification, rollback.

## (3) Work that must not be delegated

Codex should perform planning, research, spec drafting, implementation, testing, analysis, and first drafts of substantive user-facing material.

Reserved actions remain reserved:

- Claude challenges and authorizes plans; it does not create the initial substantive plan.
- The independent per-attempt reviewer remains Claude.
- Humans approve policy exceptions, privileged setup, destructive or irreversible external actions, credential use requiring operator presence, promotion from `integration` to `main`, and final trust decisions.
- Control-plane work is still planned and executed by Codex, but authorization remains Claude/human-only.
- Deterministic gates remain code, not model judgment.

Claude may edit Codex-produced prose during authorization or delivery, but a rewrite large enough to change reasoning becomes a new task requiring another Codex plan.

## (4) Failure-mode controls

Ceremony bloat is controlled through micro-plans, wrapper-generated metadata, template reuse, and one-command challenge/revision continuation. Measure planning latency; do not remove planning merely because it exposes task volume.

Plan theater is controlled by digest binding and completion reconciliation. Every completion records `followed`, `authorized deviation`, or `unauthorized deviation`. Repeated unauthorized deviations fail the process review even when tests pass.

Quota contention is managed with separate scheduler classes and weekly reserves: initially reserve 15% of Codex quota for planning/consults, 70% for workers, and 15% contingency. Planning receives low-latency queue priority but cannot preempt an active worker. Below 20% weekly quota remaining, admit only micro-plans and high-assurance work unless the human reallocates reserves.

Rubber-stamping is controlled by the mandatory written challenge and Codex disposition. For high-assurance tasks, require a second, fresh-context Codex plan critique before Claude authorization. The later Claude attempt review remains independent and must not reuse the planning authorization as evidence of correctness.

6. Enable enforcement for all substantive work, then review metrics weekly for four weeks.

Acceptance targets by week two: 100% substantive-task plan coverage; 100% plans initially drafted by Codex; at least 90% of delegable risk-weighted work performed by Codex; 100% of ordinary/high-assurance execution delegated; and at least 75% of total risk-weighted work assigned to Codex after including Claude review and authorization. Exceptions must remain below 5% and all must be human-approved and unexpired.

Rollback is component-level: if automation blocks legitimate work, disable the faulty gate through a human-approved, time-boxed exception while retaining manual detached Codex planning and artifact creation. Preserve all evidence, fix the mechanism, replay affected tasks, and re-enable the gate. Do not roll back to discretionary or planless execution.
